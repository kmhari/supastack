# Feature 116 — Operator Admin Ops Console (read-only) + Setup Docs

Read-only operator console + public CLI/MCP setup docs. Four slices: Foundation,
Resources, Queues, Cert/DNS. The mutating **Actions** slice (pause/restart/delete,
job retry, renew/backup, pooler) is **out of scope** (separate follow-up).

## Surfaces

| Path | Auth | What |
|---|---|---|
| `/docs`, `/docs/cli`, `/docs/mcp` | public | Personalized CLI wrapper + MCP setup (4 editors), copy-paste with the live apex |
| `/admin` (+ `/projects/:ref`, `/resources`, `/queues`, `/certs`, `/system`, `/logs`) | installation admin (owner/administrator) | Read-only fleet/health/resources/queues/cert observability |

All `apps/web` (React) behind a shared `AppShell`; `/setup` keeps its own layout.
Caddy routes `/docs*` + `/admin*` → `web:80` (boot `Caddyfile` + runtime
`caddy-config.ts`), before the setup-gate catch-all.

## Architecture

- **api is a pure reader.** `/api/v1/admin/*` endpoints (`apps/api/src/routes/admin.ts`)
  read DB tables + Redis (BullMQ via `queue-inspector.ts`) + delegate per-project
  health to existing platform endpoints + the per-instance `logs.all` proxy. Each
  gated by `app.authorize(req, 'admin.*')`. **No docker socket on the api.**
- **worker owns docker/host access** (Constitution V). One repeatable `observer`
  job (`apps/worker/src/jobs/observer.ts`, every `OBSERVER_INTERVAL_MS`, default
  60s) samples per-project + host resources (docker stats `--no-stream` + `df`/`du`),
  control-plane container health (`docker inspect`), and redacted recent log tails
  (`docker logs --tail`) → `resource_samples` + `control_plane_snapshots`; prunes
  older than `OBSERVER_RETENTION_DAYS` (default 7). dockerode is lazy-imported.
- **2 tables** (migration `0023_observability.sql`, idempotent, `GENERATED BY
  DEFAULT AS IDENTITY`): `resource_samples` (time-series) + `control_plane_snapshots`
  (latest-per-container).
- **RBAC**: 4 `admin.*` actions (console/resources/queues/certs) in `ADMIN_EXTRA`
  → owner + administrator only. `QUEUES.observer` in the shared constant.
- **Redaction**: `redactSensitive()` in `@supastack/shared` masks postgres URLs /
  `sbp_` / Bearer / `password=` in failed-job reasons (FR-022) and control-plane
  log tails (Constitution II).

## Key clarifications (baked in)

- Installation-wide (one operator sees all projects across all orgs).
- Admin auth reuses the existing dashboard session (`authApi.me`) — no new login.
- Logs = recent-N (project via proxy = fresh; control-plane via snapshot ≈60s).
- Capacity = used/free + average per-project footprint; **no** "N more projects" number.
- Tradeoff: control-plane data ~60s stale vs. giving the api the docker socket.

## Deploy

Rebuild `web` + `api` + `worker`; migration 0023 runs on api boot; reload Caddy
(api regenerates the config with the new routes). No new external dependency
(dockerode already present via docker-control). Env: `OBSERVER_INTERVAL_MS`,
`OBSERVER_RETENTION_DAYS` on the worker service.

## Tests
