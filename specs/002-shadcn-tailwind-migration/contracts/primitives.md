# UI Primitives Contract

**Feature**: 002-shadcn-tailwind-migration · **Phase**: 1

The complete set of shadcn primitives the migration installs, along with their public API (variants, sizes, props worth knowing), the pages that consume them, and the existing inline-style code each primitive replaces. Every primitive lives at `apps/web/src/components/ui/<name>.tsx` and is generated by `pnpm dlx shadcn@latest add <name>`.

Conventions throughout:
- All primitives export both a component and (where relevant) a `<name>Variants` `cva()` config so callers can compose.
- All primitives accept `className` and forward it through `cn()`.
- All primitives forward refs.

## button

**Path**: `apps/web/src/components/ui/button.tsx`

**Variants**:
- `default` — brand-green primary (`bg-primary text-primary-foreground hover:brightness-110`)
- `secondary` — `bg-secondary text-secondary-foreground border border-border`
- `outline` — `border border-border bg-transparent hover:bg-secondary`
- `ghost` — `bg-transparent hover:bg-secondary`
- `link` — `text-foreground-light underline-offset-4 hover:underline`
- `destructive` — `bg-destructive-bg text-destructive border border-[#5a1a1a]`

**Sizes**:
- `default` — `h-9 px-4 py-2 text-sm`
- `sm` — `h-8 px-3 text-xs`
- `lg` — `h-10 px-6`
- `icon` — `h-9 w-9 p-0`

**Replaces**: `ui.buttonPrimary`, `ui.buttonSecondary`, `ui.buttonDanger`, `ui.buttonLink` from current Shell.tsx; the green "+ New project" button in Instances toolbar; "Complete setup" in Setup; "Reveal credentials" in InstanceDetail; "Sign out" in nav.

**Consumed by**: every page.

## input

**Path**: `apps/web/src/components/ui/input.tsx`

**Props**: standard HTML input attributes via `React.InputHTMLAttributes<HTMLInputElement>`.

**Default styling**: `h-9 px-3 py-1 text-sm bg-input border border-border rounded-md focus-visible:ring-1 focus-visible:ring-ring`.

**Replaces**: `ui.input` style preset. Every `<input style={ui.input}>` becomes `<Input>`.

**Consumed by**: Setup (email, password, org name, apex domain), Login, AcceptInvite, InstancesNew (project name, database password), SettingsOrg (org name, apex, S3 fields), SettingsMembers (invite email), SettingsTokens (token label), InstanceBackups (retention number).

**Replaces**: `ui.label` + every inline `<label style={…}>` in Settings* and Setup.

**Variants**:

## Composites built on top of primitives

These live at `apps/web/src/components/*` (NOT under `ui/`), and consume `ui/*` underneath.

### `Shell`

**Path**: `apps/web/src/components/Shell.tsx`

**Public API**: `<Shell wide?>{children}</Shell>`

**Composes**: page chrome (top nav with Link tabs, user email, sign-out Button), content `<main>` with max-width container, and renders the global Toaster context.

### `PageHeader`

**Path**: `apps/web/src/components/PageHeader.tsx`

**Public API**: `<PageHeader title subtitle? right?>`

**Composes**: `<h1>` + optional subtitle paragraph + optional right-aligned slot. Used by every page.

### `StatusPill`

**Path**: `apps/web/src/components/StatusPill.tsx`

**Public API**: `<StatusPill status="running" | "paused" | "stopped" | "provisioning" | "failed" | "deleting" | "completed" />`

**Composes**: `<Badge variant={tints[status]}>`. The mapping from selfbase status string to shadcn-Badge variant lives here, in one place.

### `CopyButton`

**Path**: `apps/web/src/components/CopyButton.tsx`

**Public API**: `<CopyButton value label?="copy" size?="sm" />`

**Composes**: `<Button variant="ghost" size="sm">` + the existing secure-context-fallback logic from `Setup.tsx` (uses `navigator.clipboard` when available, falls back to `document.execCommand("copy")` with a hidden textarea for bare-IP HTTP installs). Shows "Copied ✓" feedback for 1.5s after a successful click.

### `SetupGate`

**Path**: `apps/web/src/components/SetupGate.tsx` (unchanged behaviorally; refactored to drop inline styles in favor of Tailwind classes).
