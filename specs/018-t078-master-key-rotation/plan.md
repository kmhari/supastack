## Implementation Design

### `tests/cli-e2e/t078-key-rotation.sh`

End-to-end validation script that drives the full rotation procedure programmatically. Steps:

```
Step 1   Generate NEW_MASTER_KEY (openssl rand -hex 32)
Step 2   Dry-run: node scripts/rekey-master.mjs DRY_RUN=1 в†’ assert output contains "DRY-RUN complete"
Step 3   Live run: node scripts/rekey-master.mjs в†’ assert "COMMITTED"
Step 4   Swap MASTER_KEY in infra/.env on the VM (via ssh + sed)
Step 5   Restart api + worker (via ssh + docker compose restart)
Step 6   Wait for api health check (GET /v1/profile with PAT в†’ 200)
Step 7   GET /v1/projects/:ref/api-keys в†’ assert anon_key and service_role_key present
Step 8   POST /v1/projects/:ref/pause в†’ assert status transitions to INACTIVE
Step 9   POST /v1/projects/:ref/restore в†’ poll until ACTIVE_HEALTHY (max 5 min)
Step 10  Verify: curl a request through the project's kong в†’ assert 200
Step 11  Print [T078] PASS with timestamps в†’ exit 0
```

Each step emits: `[T078] STEP: <name> | STATUS: <code_or_ok> | ELAPSED: <s>s`

**Environment variables**:
- `SUPASTACK_APEX` вЂ” e.g. `supaviser.dev`
- `SUPASTACK_PAT` вЂ” operator Personal Access Token
- `SUPASTACK_TEST_PROJECT_REF` вЂ” ref of a project to use for pause/restore test
- `SUPASTACK_VM_USER` вЂ” SSH user (default: `ubuntu`)
- `DATABASE_URL` вЂ” direct Postgres connection string for re-key tool

**Note**: Steps 4вЂ“5 (env swap + restart) require SSH access to the VM. The script uses `ssh $SUPASTACK_VM_USER@$SUPASTACK_APEX` and assumes key-based auth (same as existing operator SSH workflow).
