# Implementation Plan: Secret Reveal — No-Password UI Masking

**Branch**: `081-secret-reveal-no-password` | **Date**: 2026-05-25 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/024-secret-reveal-no-password/spec.md`

## Summary

Remove the password re-authentication gate from all three secret-reveal surfaces (JWT keys, API keys, OAuth provider client secrets). Values are fetched from the backend on demand, shown masked by default, and revealed on a single click with no dialog. The backend drops the password verification step but retains the audit log entry. A new authenticated endpoint is added for OAuth secret reveal (auth config plaintext). Frontend components across JWT Keys page, API Keys page, and six OAuth provider drawer forms are updated to the new no-dialog flow.

## Technical Context

**Language/Version**: TypeScript (Node 20 backend, React 18 + Vite frontend)

**Primary Dependencies**: Fastify 4 (API), React + TanStack Query v5 (web SPA), Drizzle ORM (Postgres), Vitest (unit tests), Playwright (E2E)

**Storage**: PostgreSQL — `project_config_snapshots` (encrypted auth config), `audit_log` (reveal events), per-project `.env` files on disk

**Testing**: Vitest (unit/integration under `packages/` and `apps/`), Playwright Chromium (E2E under `apps/web/tests/e2e/`)

**Target Platform**: Linux server (Fastify API in Docker) + modern browser (React SPA served via Caddy)

**Project Type**: Monorepo web application — Fastify API (`apps/api`) + React SPA (`apps/web`)

**Performance Goals**: Reveal API call resolves in <2 seconds p95 under normal conditions

**Constraints**: No password in any reveal request body; audit log entry on every reveal regardless of surface; reveal is admin-only (RBAC gate)

**Scale/Scope**: Single-operator platform; typically <10 admin users per instance; no high-concurrency concern for reveal endpoints
