# Implementation Plan: Shared Studio (IS_PLATFORM=true)

**Branch**: `083-shared-studio-platform` | **Date**: 2026-06-01 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/025-shared-studio-platform/spec.md`

## Summary

Replace N per-project Studio containers with a single shared Supabase Studio service running in the control plane at `IS_PLATFORM=true`, served at the apex root (`https://<apex>/`). Studio's client-side data fetching already routes all `/platform/*` requests to `NEXT_PUBLIC_API_URL` вЂ” it does not use the Next.js `/api/*` server routes in platform mode (verified by tracing `apps/studio/data/fetchers.ts` on the upstream Studio repo). Supastack's Fastify API gains platform-proxy routes that resolve a project `ref` to its `portKong` and pipe requests to the appropriate per-instance Kong gateway.

## Technical Context

**Language/Version**: TypeScript 5.x (Node 20), Docker Compose v2

**Primary Dependencies**: Fastify (`apps/api`), undici (HTTP proxying), React + TanStack Query (Studio SPA), Playwright (e2e), Vitest (unit)

**Storage**: `port_allocations` table вЂ” `portKong` column (already populated per instance by the provision pipeline)

**Testing**: Vitest unit tests for proxy route resolution; Playwright e2e smoke for Studio navigation

**Target Platform**: Linux Docker (single VM deployment at `ubuntu@148.113.1.164`)

**Project Type**: Platform service migration + new API proxy layer

**Performance Goals**: Proxy hop adds < 50ms overhead over direct Kong access; Studio pages load within 3s under normal conditions

**Constraints**:
- Zero modifications to Supabase Studio source code
- `NEXT_PUBLIC_*` env vars are baked at Next.js build time вЂ” Studio image requires rebuild per apex domain
- Proxy routes must use existing `app.authenticate` plugin
- Must not forward `x-connection-encrypted` header to upstream services
- Must strip upstream CORS headers (Kong/pg-meta return their own, causing conflicts)

**Scale/Scope**: 1 shared Studio container; ~4 new proxy route groups; 2 compose file edits; 1 Caddy config edit

## Phase 0: Research

*Resolved inline вЂ” no external unknowns requiring agent dispatch.*

Key decisions:

| Decision | Choice | Rationale |
|---|---|---|
| HTTP proxy mechanism | `undici` (already a Node.js built-in dep) or `@fastify/reply-from` | `@fastify/reply-from` is the idiomatic Fastify proxy; handles streaming, headers, and body correctly. Check if already present in `apps/api/package.json`; if not, prefer raw `undici.pipeline` to avoid adding a dep. |
| Studio image strategy | `next dev` in container (Phase 1) | Avoids custom Docker build and `NEXT_PUBLIC_*` bake complexity for initial ship. Production-optimised build is a Phase 2 follow-up. |
| Studio URL | `https://<apex>/` (apex root) | No subdomain needed; wildcard cert already covers apex; Caddy specificity routes `/api/v1/*` and `/setup*` before `/*`. |
| Auth for Studio | Existing PAT / GoTrue session (feature 011 flow) | Studio's `IS_PLATFORM=true` sign-in page calls `NEXT_PUBLIC_GOTRUE_URL` which is the platform API вЂ” reuses the existing Supastack auth stack. |
| `portStudio` handling | Leave column, stop allocating | Non-destructive; avoids migration complexity; existing rows are harmless. |

See [research.md](research.md) for full proxy library comparison.

## Phase 1: Design & Contracts

### Platform Proxy Route Design

All proxy routes follow the same pattern:

```
1. app.authenticate(req)          вЂ” 401 if unauthenticated
2. db lookup: portKong for ref    вЂ” 404 if ref unknown
3. undici/reply-from forward      вЂ” strip x-connection-encrypted, strip upstream CORS headers
4. pipe response back             вЂ” include body streaming for large results
```

Route groups:

| Fastify route | Upstream (Kong) | Notes |
|---|---|---|
| `GET/POST/PATCH/DELETE /platform/pg-meta/:ref/*` | `http://localhost:<portKong>/pg-meta/v0/*` | Strip `x-connection-encrypted` |
| `GET/POST/PATCH/DELETE /platform/storage/:ref/*` | `http://localhost:<portKong>/storage/v1/*` | Full REST passthrough |
| `GET/POST/PATCH/DELETE /platform/auth/:ref/users*` | `http://localhost:<portKong>/auth/v1/admin/users*` | Admin auth ops |
| `POST /platform/auth/:ref/invite` | `http://localhost:<portKong>/auth/v1/admin/users` | Invite = create |
| `POST /platform/auth/:ref/magiclink` | `http://localhost:<portKong>/auth/v1/admin/generate_link` | |
| `POST /platform/auth/:ref/otp` | `http://localhost:<portKong>/auth/v1/admin/generate_link` | |
| `POST /platform/auth/:ref/recover` | `http://localhost:<portKong>/auth/v1/admin/generate_link` | |
| `GET/POST /platform/projects/:ref/analytics/*` | `http://localhost:<portKong>/analytics/v1/*` | |

### Caddy Routing

The apex site block gains one rule (lowest priority, last match):

```caddy
handle /* {
  reverse_proxy studio:3000
}
```

Existing `/api/v1/*` and `/setup*` blocks remain unchanged and match first due to Caddy's specificity ordering.

### Studio Container (dev-server phase)

```yaml
studio:
  image: node:20-alpine
  working_dir: /studio
  command: pnpm --filter studio exec next dev -p 3000
  volumes:
    - /opt/supabase-vanilla:/studio:ro
  environment:
    NEXT_PUBLIC_IS_PLATFORM: "true"
    NEXT_PUBLIC_API_URL: "https://${APEX_DOMAIN}/api/v1"
    NEXT_PUBLIC_GOTRUE_URL: "https://${APEX_DOMAIN}/api/v1"
  restart: unless-stopped
```
