# Data Model — Feature 116 (Admin Ops Console + Docs)

Two **new** tables (idempotent, additive migrations). Everything else is read over existing entities.

## New: `resource_samples` (time-series)

Per-tick measurements for trends + current values. Pruned to ~7 days.

| Column | Type | Notes |
|---|---|---|
| `id` | bigint, `GENERATED BY DEFAULT AS IDENTITY` PK | Surrogate — NOT `SEQUENCE … OWNED BY` (migrate.ts owner gotcha). |
| `captured_at` | timestamptz NOT NULL | Tick time (worker-supplied; `now()` default ok). |
| `scope` | text NOT NULL | `'host'` or `'project'`. CHECK constraint (widen by drop+recreate if extended). |
| `ref` | text NULL | Project ref when `scope='project'`; NULL for host. |
| `cpu_pct` | numeric NULL | Aggregate CPU % (sum of the scope's containers). |
| `mem_used_bytes` | bigint NULL | Memory in use. |
| `mem_limit_bytes` | bigint NULL | Memory limit (host total for `scope='host'`). |
| `disk_used_bytes` | bigint NULL | Project data dir size (project) / total used (host). |
| `disk_breakdown` | jsonb NULL | Host-only: `{ project_data, backups, other, free }`. |

Indexes: `(scope, ref, captured_at desc)` for "latest per project" + trend queries; `(captured_at)` for prune.

**Lifecycle**: insert-only per tick; prune deletes `captured_at < now() - retention`. No updates.

## New: `control_plane_snapshots` (latest-per-container)

One row per control-plane container, upserted each tick. Bounded (≈8 rows).

| Column | Type | Notes |
|---|---|---|
| `container` | text PK | e.g. `supastack-api-1`. |
| `captured_at` | timestamptz NOT NULL | Last refresh. |
| `health` | text NULL | `healthy` / `unhealthy` / `starting` / `none` (from `docker inspect`). |
| `status` | text NULL | e.g. `Up 44 hours`. |
| `image` | text NULL | Image:tag (version signal). |
| `log_tail` | text NULL | Last N lines, **secret-redacted** before write. |

**Lifecycle**: `INSERT … ON CONFLICT (container) DO UPDATE` each tick. No history.

## Read-only views over existing entities (no new storage)

- **Project (fleet)** — `supabase_instances` (ref, name, org_id, status, created_at, ports). Joined to `organizations` for org name. Installation-wide (all rows).
- **Per-project health** — existing `/platform/projects/:ref/{status,services,service-versions,databases-statuses}` (reused server-side).
- **Background-work item** — BullMQ runtime state in Redis (not a table): `getJobCounts()` + `getFailed()` per `QUEUES.*`. Failed item = `{ id, name, failedReason (redacted), failedAt, attemptsMade }`.
- **Certificate** — `wildcard_certs` (`apex`, `not_after`, status) + `pg_edge_certs` (per-project expiry). Days-left derived; renewal warning if `not_after < now()+30d`.
- **DNS readiness** — apex/wildcard A-record + ACME `allDnsReady` (feature 087 signal).
- **Backup record** — backups store + per-project last-backup (time/size/outcome); total storage from `control-plane`/host `disk_breakdown.backups`.

## RBAC additions (`packages/shared/src/rbac.ts`)

New `ACTIONS` entries, granted to `owner` + `administrator` only (false for `developer`, `read_only`):

- `admin.console.read` — list fleet, project health, system status, logs.
- `admin.resources.read` — resource samples + capacity.
- `admin.queues.read` — queue/job health.
- `admin.certs.read` — cert/DNS/backup status.

(Single-action `admin.console.read` is acceptable if the team prefers coarse gating; the granular set is the default for per-tab auditability. The matrix contract test must cover every new cell.)

## Queue name addition (`packages/shared/src/queues.ts`)

- `QUEUES.observer` (e.g. `'supastack.observer'`) — the repeatable resource/health/log sampler. Produced + consumed only by the worker; added to the shared constant per Constitution V (no literal drift).
